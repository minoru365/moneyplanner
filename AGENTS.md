# Codex Instructions

毎タスク開始時に [CLAUDE.md](CLAUDE.md) と [.github/copilot-instructions.md](.github/copilot-instructions.md) を必ず両方読む。2ファイルは同格の正典として扱い、片方だけで判断しない。内容に矛盾がある場合は、作業前にユーザーへ報告する。

## モデルハンドオフ

- タスク開始時に、計画のみ・読み取りのみ・コード変更ありのいずれかに分類し、編集前にモデルハンドオフを確認する。
- 計画・設計には `gpt-5.6-sol` を使う。親エージェントがSolでなければ、計画工程をSolサブエージェントへ委譲する。
- コード変更を伴うタスクの実装は、`model: "gpt-5.6-terra"`、`fork_context: false` の `worker` サブエージェントへ委譲し、必要な背景と要件を委譲プロンプトへ明記する。
- Terraには担当ファイルまたはモジュールを明示し、作業ツリーに他者の変更が存在し得ること、および他者の変更を戻さないことを指示する。
- 親エージェントはTerraの変更をレビューし、必要な修正を統合して、関連テストとlintを実行し結果を報告する。
- 実装前に、指定モデルでTerraサブエージェントが実際に起動したことを確認する。指定モデルを起動していない工程をTerra作業と表現しない。
- SolまたはTerraが利用できない場合は、黙って代替せず実装前にユーザーへ報告する。
- 最終回答ではSolとTerraの実呼び出し回数を報告する。ツール出力がない限り、トークン使用量や費用を推測しない。
- 読み取りのみの確認、状況報告、直接の質問は、コード変更につながらない限りTerraハンドオフを必要としない。
