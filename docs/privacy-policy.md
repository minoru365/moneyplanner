# moneyplanner プライバシーポリシー（草案）

最終更新日: 2026-05-10

本ポリシーは、moneyplanner（以下「本アプリ」）におけるユーザーデータの取り扱いについて定めるものです。

## 1. 取得する情報

本アプリは、以下の情報を取得・保存します。

- 認証情報
  - Firebase Auth の `uid`
- アプリ利用情報
  - 世帯ID（`householdId`）
  - 表示名（`displayName`）
- 家計簿データ
  - 取引（日時、金額、種別、カテゴリ、内訳、口座、店舗名、メモ等）
  - カテゴリ、内訳、予算、口座、店舗情報
  - 世帯メンバー情報、招待コード情報

補足:

- Apple Sign-In 由来のメールアドレス・氏名は、アプリDBへ恒久保存しません。
- 招待コードは参加導線のために利用し、有効期限や無効化状態を管理します。

## 2. 利用目的

取得した情報は、以下の目的でのみ利用します。

- 家計簿の記録、集計、履歴表示
- 世帯内リアルタイム同期
- 世帯参加・招待コード運用
- CSVエクスポート機能の提供
- 不正利用の抑止（例: 招待コード試行回数制御）

## 3. 共有・第三者提供

- 家計簿データは、同一世帯に参加しているメンバー間でのみ共有されます。
- 開発・分析・広告・外部提供を目的とした二次利用は行いません。
- 法令に基づく場合を除き、ユーザーの同意なく第三者へ提供しません。

## 4. 保存場所・アクセス制御

- データは Firebase Cloud Firestore に保存されます。
- Firestore Security Rules により、自世帯メンバー以外の読み書きを禁止します。
- 解除済みメンバーは世帯データへアクセスできません。

## 5. 保存期間

- ユーザーが削除するまで、家計簿データを保存します。
- 世帯データ削除またはアカウント削除操作が行われた場合、該当データは削除対象になります。

## 6. ユーザーによる削除

本アプリでは、設定画面から次の削除操作が可能です。

- 認証解除と全データ削除
  - Firebase Authアカウント削除
  - 当該ユーザーの `/users/{uid}` 削除
  - 世帯データ削除（共有データを含む）
- メンバー解除
  - 世帯から特定メンバーを解除

注意:

- 世帯データ削除は、他メンバーにも影響します。
- ローカルキャッシュは端末状態により即時消去されない場合がありますが、次回起動時の権限チェックで利用不可になります。

## 7. セキュリティ対策

- Firebase Authentication による認証
- Firestore Security Rules によるデータアクセス制御
- Firebase App Check による不正利用対策
- 招待コードの有効期限・無効化、および試行回数制御

## 8. 未成年の利用

本アプリは年齢を問わず利用できますが、法定代理人の同意が必要な場合は、利用者側の責任で同意を得てください。

## 9. ポリシーの改定

本ポリシーは、法令・運用・機能変更に応じて改定することがあります。重要な変更がある場合は、アプリ内または配布ページで案内します。
