# ローカル生成物の Git 除外設計

## 目的

App Review 用動画などのローカル生成物と、検証時の一時ファイルがリポジトリへ混入しないようにする。

## 対象範囲

- 変更対象: `.gitignore`
- 除外対象: `artifacts/` 全体、`.codex-tmp/` 全体
- 対象外: 既存ファイルの削除、アプリコード・設定・依存関係の変更

## 検討案

1. `artifacts/` と `.codex-tmp/` をディレクトリ単位で除外する（採用）
2. `artifacts/review/` と `.codex-tmp/` のみ除外する
3. `*.mp4` など拡張子単位で除外する

採用案は、今後増える生成物も一貫して除外でき、正規の動画素材を拡張子だけで誤って隠すリスクがない。

## 受入条件と検証

- `.gitignore` に `artifacts/` と `.codex-tmp/` が記載されている
- `git check-ignore -v` で両ディレクトリ配下が無視対象になる
- `git status --short` に `artifacts/` が未追跡として表示されない
- 既存の App Review 用動画はローカルに残る

## リスク

`artifacts/` 配下へ将来コミット対象を置いても既定では追跡されない。共有が必要な成果物は別ディレクトリに置くか、明示的に除外ルールを見直す。

変更は Git の追跡対象だけに影響し、実行時機能やデータには影響しないため、独立監査は不要とする。
