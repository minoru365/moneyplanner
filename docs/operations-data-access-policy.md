# 本番データアクセス運用ルール

最終更新日: 2026-06-30

## 目的

ユーザーの家計明細（取引金額、メモ、店舗名、カテゴリ等）を、開発・運用において日常的に閲覧しないことを徹底する。

## 基本方針

- 開発・運用で日常的に使う環境は、開発/検証データのみとする
- 本番Firestoreのユーザー家計明細は、通常業務で閲覧しない
- 本番データを分析・広告・外部提供目的で二次利用しない
- AI/外部ツールへ本番の家計明細・認証情報を渡さない

## 本番データ参照を許可する例外条件

本番データの参照は、以下をすべて満たす場合のみ許可する。

1. 問い合わせ対応や重大障害調査など、代替手段では原因特定できない
2. 参照範囲を最小化できる（対象ユーザー/対象期間/対象項目）
3. ユーザー同意が必要な場合は、事前に同意を取得する
4. 実施記録を残す

## ユーザー同意と記録

問い合わせ調査で本番データの参照が必要になった場合は、参照前に以下をユーザーへ説明し、明示的な同意を得る。

- 調査目的
- 参照する可能性があるデータ範囲（例: 対象期間、対象機能、対象世帯）
- 参照しないデータ（調査に不要な取引明細、認証情報、秘密情報など）
- 調査後に個票データを保存しないこと
- 同意しなくても通常サポートを可能な範囲で継続すること

同意と作業記録には、同意日時、対象範囲、担当者、作業開始/終了時刻、結果概要、削除/匿名化した内容を残す。記録には金額・メモ・店舗名などの家計明細をそのまま書かない。

## 参照時の実施手順

1. 参照目的、対象、想定作業時間、担当者を記録する
2. 必要最小限の範囲のみ参照する
3. 金額・メモ・店舗名などの明細を外部へ転記しない
4. 調査結果は匿名化し、個票データを残さない
5. 作業終了後、記録を更新してクローズする

## ログ・共有時の禁止事項

- チャット、Issue、PR、ログに家計明細を貼り付けない
- スクリーンショットで個人情報/明細が読める状態で共有しない
- Firebase secret、認証情報、トークンを記録しない
- サポート用ログ、クラッシュログ、問い合わせ返信には、取引金額、メモ、店舗名、口座名、個人を推測できる世帯名/表示名を含めない
- 再現説明が必要な場合は、金額を丸めた例、匿名化したカテゴリ名、テストデータで置き換える

## 権限管理

- Firebase Console / GCP IAM は最小権限原則で付与する
- 本番アクセス可能アカウントは最小人数に限定する
- 不要になった権限は即時剥奪する
- 日常開発用アカウントには本番Firestore閲覧権限を付与しない
- 本番データを参照できる権限は、請求/リリース/障害対応に必要な管理者アカウントに限定する
- 本番アクセス可能アカウントはリリース前、メンバー変更時、少なくとも四半期ごとに棚卸しする
- 調査用に一時権限を付与した場合は、作業終了後すぐに剥奪し、作業記録に残す
