# 計画タブ廃止

## 決定

計画タブおよび関連する Firestore 保存データ（`planProfile` / `planLifeEvents`）とシミュレーション補助コードを、現行実装から削除する。

## 背景

計画機能は実装と運用の両面で負荷が大きく、現時点では利用頻度に対して維持コストが高いと判断した。
画面、Firestore、共通ロジック、関連ドキュメントまで含めると変更面が広く、他のコア機能の改善を遅らせる要因にもなっていた。

## 影響範囲

- `app/(tabs)/plan.tsx` を削除
- `lib/simulation/*` を削除
- Firestore の `planProfile` / `planLifeEvents` を削除対象から外し、参照経路を除去
- 画面定義、README、ARCHITECTURE、PLAN、既知課題、AI運用ルールを整合化

## 復元方針

将来必要になった場合は、Git 履歴から復元できる前提とする。
復活する際は、次の条件を満たしてから再導入を検討する。

- 家族内で実際に使う明確なニーズがある
- 維持対象の範囲が縮小されている、または別機能へ統合できる
- Firestore 保存、同期、削除フロー、ドキュメント更新まで一括で戻す作業計画がある
- 既存のコア機能より優先度が高いと判断できる

## 判断メモ

- 1回削除しても Git 履歴で戻せるため、今は「維持しない」判断を優先する
- 復元時は、画面だけでなく DB・削除フロー・テスト・ドキュメントを同時に戻す必要がある
